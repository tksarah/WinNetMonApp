# 通信状況確認アプリ

Version: 2026.3.14（バージョンは日付で管理）

Windows 11 で「いつも見ているWebページにアクセスできない」時の切り分け用アプリです。
困ったときに「診断開始」を押して、画面の内容をスマホで写真をとり、健史にLINEで送ってください。

## 目的（できること）

- 最大5件のURLを登録し、サイトごとに到達可否をチェックします
- PC側の基本的な通信状況（ネットワークアダプタ / IP / DNS / プロキシ）を表示します
- 結果は画面に表示するだけ（レポート保存・常駐監視はしません）

## 使い方（利用者向け）

1. 画面上の「サイト1〜5」に、見たいURLを入力します（例: `https://example.com/`）
2. 「保存」を押します（次回起動時も残ります）
3. 「診断開始」を押します
4. 表示された結果を、スマホで撮影してLINEで送ります

### 表示される結果（仕様）

画面は概ね次の3つに分かれます。

#### 1) 基本情報

- 日時: 診断を実行した時刻
- プロキシ: Windowsのプロキシ設定（OFF / ON / PAC / 不明）

#### 2) 通信確認

- ネットワークアダプタ: 主要アダプタが有効かどうか（OK/NG）
- IPアドレス: 取得できているか（OK/NG + 表示）
- DNS名前解決（google.com）: DNSが引けるか（OK/NG）

#### 3) サイト診断（登録URLごと）

- OK/NG と、理由（例: DNS失敗 / タイムアウト / TLSエラー / HTTPエラー など）

## 画面・操作の仕様

- URL登録: 最大5件、画面で編集して「保存」します（ローカル保存）
- 入力の「削除」: その行のURLだけ消します
- 文字サイズ: 右上の「小/標準/大/特大」で切替（ローカル保存）
- 起動位置: 画面の「中央の上の方」に出るようにしています
- 診断後: 内容に合わせてウィンドウを可能な範囲で拡大します（収まりきらない場合は最大化します）

## データの扱い（大事）

- このアプリはバックグラウンド常駐しません
- 入力したURLと文字サイズ設定は、このPC内（ローカル）に保存されます
- 診断のために、登録URLへのアクセス確認と `google.com` の名前解決確認を行います

## 開発（Windows / PowerShell）

PowerShellの実行ポリシー設定によっては `npm`（npm.ps1）が実行できないため、`npm.cmd` を使います。

- `npm.cmd install`
- `npm.cmd run tauri dev`

ビルド:

- `npm.cmd run build`
- `npm.cmd run tauri build`
